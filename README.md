## 1. 개요

### 1.1 비교 목적
동일한 자동화 워크플로우를 Make와 Zapier 두 도구로 각각 구현하여,
무료 플랜 기준의 기능·사용성·비용 차이를 실제 경험 기반으로 비교 분석한다.

### 1.2 구현한 워크플로우
**"퀴즈 응시 결과 자동 분류 및 알림"**

```
[Trigger] Google Form 제출 → 응답 Sheet에 새 행 추가 감지
    ↓
[분기] 점수 ≥ 80 ?
    ├─ Yes → 합격자 시트에 기록 → Discord 합격 알림 🎉
    └─ No  → 불합격자 시트에 기록 → Discord 불합격 알림 📋
```

- 구성 요소: Trigger 1개 + 조건 분기 1개 + Action 4개
- 연동 서비스: Google Sheets, Discord

## 2. 구현 방식 비교

### 2.1 전체 구조 차이 ⭐ (가장 큰 차이점)

| 항목 | Make | Zapier (무료) |
|---|---|---|
| 워크플로우 개수 | **시나리오 1개** | **Zap 2개** (합격용 + 불합격용) |
| 분기 처리 방식 | Router 모듈로 시각적 분기 | Filter로 조건 미충족 시 중단 → Zap을 분리해야 분기 구현 가능 |
| 분기 기능 제공 여부 | 무료 플랜 포함 | Paths(분기)는 유료(Professional) 기능 |

**체감 소감**: [예: Make는 한 화면에서 전체 흐름이 보여서 관리가 편했다. Zapier는 같은 로직인데 Zap 2개를 만들어야 해서 수정 시 두 번 고쳐야 하는 번거로움이 있었다.]

### 2.2 Discord 연동 방식 차이

| 항목 | Make | Zapier (무료) |
|---|---|---|
| 사용한 방식 | HTTP 모듈 + Webhook URL | Discord 공식 앱 (봇 초대) |
| 설정 난이도 | [예: JSON 바디를 직접 작성해야 함] | [예: 봇 초대 후 채널만 선택하면 됨] |
| 유연성 | 자유로운 HTTP 요청 가능 (무료) | Webhooks by Zapier는 프리미엄(유료) 앱 |

## 3. 사용성 비교

### 3.1 UI/UX

| 항목 | Make | Zapier |
|---|---|---|
| 편집 방식 | 캔버스 기반 (모듈을 자유 배치, 흐름이 그림으로 보임) | 세로 스텝 리스트 (위→아래 순차 나열) |
| 학습 곡선 | [예: 처음엔 낯설지만 익숙해지면 직관적] | [예: 처음 접해도 바로 이해되는 구조] |
| 데이터 매핑 | 말풍선에서 필드 드래그 | 드롭다운 + 검색으로 필드 삽입 |
| 테스트 편의성 | Run once로 전체 흐름 한 번에 실행, 모듈별 입출력 확인 | 스텝별 개별 Test, 전체 흐름 테스트는 상대적으로 번거로움 |

### 3.2 실행 로그/디버깅

- **Make**: [예: History에서 실행별로 각 모듈의 입출력 데이터를 상세히 볼 수 있어 디버깅이 쉬웠다]
- **Zapier**: [예: Zap History에서 성공/Filtered/실패 상태가 한눈에 보이고, Filtered 사유도 확인 가능했다]

### 3.3 구현 중 겪은 문제와 해결

| 문제 | 원인 | 해결 |
|---|---|---|
| [예: Router 필터에서 60점이 합격 처리됨] | [Text 연산자로 문자열 비교] | [Numeric 연산자로 변경] |
| [겪은 문제 추가] | | |

## 4. 무료 플랜 비용/제한 비교

| 항목 | Make Free | Zapier Free |
|---|---|---|
| 월 실행 한도 | 1,000 Operations | 100 Tasks |
| 과금 단위 | Operation = 모듈 1회 실행 | Task = Action 1회 실행 (Trigger/Filter는 미차감) |
| 이번 프로젝트 1회 실행 소모량 | 약 [예: 4] Operations (Trigger+Router+Action 2) | 약 2 Tasks (통과한 Zap의 Action 2개) |
| 분기(멀티 경로) | ✅ 무료 | ❌ Paths는 유료 |
| HTTP/Webhook 모듈 | ✅ 무료 | ❌ 프리미엄 앱 |
| 최소 폴링 간격 | 15분 | 15분 |
| 실제 사용량 (테스트 후) | [예: 12 / 1,000 Ops] | [예: 8 / 100 Tasks] |

**분석**: [예: 단순 환산 시 Make가 실행 한도 면에서 훨씬 여유롭다. 다만 Make는 모듈 단위 과금이라 복잡한 시나리오일수록 소모가 빨라진다. 이번 워크플로우 기준 월 처리 가능 건수는 Make 약 250건, Zapier 약 50건 수준.]
